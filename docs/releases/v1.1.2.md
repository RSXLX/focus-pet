# Focus Pet v1.1.2

发布状态：已发布

## 概要

v1.1.2 是更新体验和 Cloud 聊天 patch release。它把“发现新版本后打开 GitHub Release 页面”的流程改为“宠物气泡和系统通知提醒，用户确认后直接下载并打开最新安装包”，同时补齐 Cloud 文字/图片消息和更易用的 6 位数字好友码。

## 变更

- 更新检查继续使用 GitHub Release latest feed，并优先选择当前平台最合适的 DMG/ZIP 资产。
- 自动检查发现新版本时，桌宠会显示“新”气泡提示；点击气泡会直接下载并打开安装包。
- 系统通知点击后也会直接下载最新安装包，不再要求用户进入 Release 页面手动找文件。
- 手动“检查更新”发现新版本时，会直接下载并打开安装包。
- 下载目标为系统 Downloads 目录，文件名来自 Release asset，并自动避免覆盖已有文件。
- 应用仍不会静默替换自身或自动重启安装包，安装动作由用户确认。
- Cloud 好友码改为 6 位数字，并在桌面端提供一键复制。
- Cloud 好友支持发送文字和图片；图片当前限制为 5MB 以内。
- Cloud 通话按钮点击时会主动连接 WebSocket，不再因为刚打开面板时 socket 未连接而无法发起。
- 设置、引导和复盘切到更大的客户端面板；悬浮桌宠顶部只保留消息、任务、设置、收起和退出等核心入口。
- 发布清单已对齐当前 ad-hoc signed 路径：`release:preflight -- --run package` 默认生成 `release:mac` 资产，Developer ID 签名和 notarization 保留为后续人工可选项。

## 签名状态

本版本按用户要求不做 Apple Developer ID 签名和 notarization。macOS 包会进行 ad-hoc signing，但不是 Apple-notarized build。首次启动时，macOS Gatekeeper 可能要求在“系统设置”中手动允许，或通过右键“打开”启动。

## 注意

- 这不是后台静默自更新。当前未做 Apple notarization，不适合在后台直接替换正在运行的 App。
- 如果未来要做到更接近原生无感更新，需要引入 Developer ID 签名、公证和专门的 updater 差分/替换链路。
